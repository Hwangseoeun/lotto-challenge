## 전체 조건
- 사용자가 잘못된 값을 입력할 경우 ```IllegalArgumentException```을 발생
-> 에러 메시지 출력 후 그 부분부터 다시 입력받음


## 기능 정리
- 사용자로부터 로또 구입 금액을 입력받는 기능
	- 조건 : 1,000원 단위로 입력
	- 에러 : ```IllegalArgumentException``` 발생
        1. "[ERROR] 구입 금액은 1,000원 단위로만 가능해요."


- 구입 금액만큼 발행해야 하는 로또 수량을 계산하는 기능
    - 조건 : 로또 1장의 가격은 1,000원


- 로또 번호 6개를 랜덤으로 뽑아 로또를 발행하는 기능
    - 조건 :
        1. 1 ~ 45 사이의 숫자
        2. 중복되지 않는 로또 번호 6개로 구성
        3. 로또 번호는 오름차순으로 정렬
    - 에러 : ```IllegalArgumentException``` 발생
        1. "[ERROR] 로또 번호는 1 ~ 45 사이의 숫자만 가능해요."
        2. "[ERROR] 로또 번호는 중복되면 안되며 번호는 6개여야 해요."


- 발행한 로또 수량만큼 로또를 출력하는 기능


- 사용자로부터 당첨 번호를 입력받는 기능
    - 조건 :
        1. 쉼표를 기준으로 구분
        2. 번호는 1 ~ 45 사이의 숫자
        3. 중복되지 않는 숫자 6개
    - 에러 : ```IllegalArgumentException``` 발생
        1. "[ERROR] 로또 번호는 1 ~ 45 사이의 숫자만 가능해요."
        2. "[ERROR] 로또 번호는 중복되면 안되며 번호는 6개여야 해요."
        3. "[ERROR] 당첨 번호는 ,를 기준으로 구분해야 해요."


- 사용자로부터 보너스 번호를 입력받는 기능
    - 조건 :
        1. 번호는 1 ~ 45 사이의 숫자
        2. 뽑은 당첨 번호와 중복 X
    - 에러 : ```IllegalArgumentException``` 발생
        1. "[ERROR] 보너스 번호는 1 ~ 45 사이의 숫자만 가능해요."
        2. "[ERROR] 당첨 번호와 중복되는 숫자는 입력할 수 없어요."


- 구매한 로또와 당첨 번호가 몇개 일치하는지 판단하는 기능


- 구매한 로또와 보너스 번호가 일치하는지 판단하는 기능


- 구입한 로또들이 각각 몇등 당첨인지 판단하는 기능
    - 조건 : 구입한 모든 로또에 대한 판단


- 구입한 로또들 중 몇개가 당첨되었는지 당첨 통계를 출력하는 기능
    - 조건 : 구입한 모든 로또에 대한 통계


- 수익률을 계산하는 기능
    - 계산식 : (수익 금액 / 구입 금액) * 100


- 구입한 로또에 대한 수익률을 출력하는 기능
    - 조건 : 소수점 둘째 자리에서 반올림 (ex. 51.5%)
